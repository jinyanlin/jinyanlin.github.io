Vue3學習紀錄 DAY3

vue
Vue3
Publish Date:   2024-11-08

指令

帶有 v- 開頭的屬性，在 Vue.js 裡面就被稱為 「指令」(directive) ， 當與指令搭配運算式 (expression) 的值被改變時，對應的標籤也會隨之影響。

屬性綁定 - v-bind

<body>
    <div id="app">
        <p id="{{ customid }}">...</p>
    </div>
</body>
<script>
     const app = Vue.createApp({
            data() {
                return {
                        customid: 'mycustom'
                };
            },
        }).mount('#app');
</script>

上面程式的執行結果，標籤 <p> 的 id 並不會如我們所預期地出現 mycustom， 因為像 { } 的模板語法並沒有辦法被套用在 HTML 標籤的「屬性」上， 如果我們需要由 Vue.js 來控制標籤的「屬性」，則可以使用 v-bind 的指令。

v-bind 指令的用法很簡單，我們只要再標籤上面加上v-bind:屬性名稱， 以剛剛的範例來說，我們希望由 Vue 的實體來輸出 id，就可以這樣做：

<p v-bind:id="customid">...</p>

實際在瀏覽器渲染時會變成：

小提醒

v-bind 指令除了完整的 v-bind:屬性名稱 寫法外，也可以直接簡寫成 :屬性名稱，如：

<p v-bind:id="customid">...</p>


<p :id="customid">...</p>

表單綁定 - v-model

常見的表單元素像是 <input>、<textarea> 以及 <select> 等，在 Vue.js 可透過 v-model 來進行資料的「雙向」綁定， v-model 會根據不同的表單類別來更新元素的內容。

input 文字框

當我們在 input 加上 v-model="text" 之後，此時這個輸入框便會自動被綁定 input 事件。


如果我們希望輸入文字框時可同步顯示在下面，可以這樣做 : 
<div id="app">
    <input class="form-control" v-model="text">
    <div v-text="text"> </div>
</div>

<script>
    const app = Vue.createApp({
        data() {
            return {
                customid: 'mycustom',
                text: '',
            };
        },
    }).mount('#app');
</script>

radio / checkbox 選擇框

input 標籤提供了兩種選擇框，分別是 radio 的單選框，以及 checkbox 的複選框。


radio 的單選框我們只需要在對應的 input 加上 v-model，並且透過 value 指定它的值：

<div id="app">
    <div>
        <input type="radio" id="one" value="1" v-model="picked">
        <label for="one">One</label>
    </div>
    <div>
        <input type="radio" id="two" value="2" v-model="picked">
        <label for="two">Two</label>
    </div>
    <span>Picked: {{picked}} </span>
</div>

<script>
    const app = Vue.createApp({
        data() {
            return {
                customid: 'mycustom',
                text: '',
                picked:1
            };
        },
    }).mount('#app');
</script>

因為 data 裡的 picked 預設為 1，所以執行時畫面上 <input type="radio" id="one" value="1"> 會預設被選中。

select 下拉列表選單

select 元素是很常見的下拉式選單，它用來與 option 做搭配，如下：

<div id="app"> 
    <select v-model="selected">
        <option disabled value="">請選擇</option>
        <option>Lin</option>
        <option>Chen</option>
        <option>Huang</option>
    </select>
    <p>Selected: {{ selected || '未選擇' }}</p>
</div>
const app = Vue.createApp({
    data(){
        return {
            selected: '',
        }
    }
}).mount('#app');

v-model 必須使用在 <select> 標籤，不可套用在 <option> 標籤中。

v-model 與修飾子

v-model 在 <input> 的 text 文字輸入框對應的是 input 事件。

當使用者每敲一次鍵盤就會觸發一次 input 事件，若是不希望這個文字輸入框的內容更新次數這麼頻繁，
Vue.js 提供了「修飾子」 (Modifiers) 的增強功能來協助我們，修飾子會以 .XXX 的形式附加在指令後面。

.lazy
<input type="text" v-model.lazy="message">

v-model後面加上.lazy，這個input輸入框就會從原本input事件改為監聽change事件，明白地說，使用者在離框輸入焦點時才會更新data內容。

.number

在 input 輸入框當中，即便我們輸入的都是數字，但實際上由 DOM API 取出的時候都會是以「字串」的形式出現。

當使用者分別為 num1 與 num2 兩個輸入框填入 1 與 2 後， sum 仍會出現 12 的結果。

<input v-model.number="num1"> + <input v-model.number="num2"> = {{ sum }}

如此一來，當 v-model 在更新狀態的時候，會在背後試圖將對應的資料轉型為數字的格式， 即便 sum 維持 return this.num1 + this.num2 的程式碼，回傳的也會是兩個數字相加的結果了。

.trim

想要針對某個輸入框自動過濾前後的空白字元時，就可以在 v-model 後面加上 .trim 修飾子：

<input v-model.trim="message">
